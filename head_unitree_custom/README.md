# Управление умной головой Кузьмича — Light (LED + Servo)

Прошивка на базе [`esp32-control-c3-light`](https://github.com/mIbkola/esp32-control-c3-light)
для платы **ESP32-C3** (DevKitM-1). Аудио-модуль в проекте отсутствует.

## Железо

| Сигнал        | Пин ESP32-C3 | Примечание                          |
|---------------|--------------|-------------------------------------|
| WS2812B DIN   | GPIO10       | 2 панели × 7 = **14** адресных LED   |
| Servo 1 PWM   | GPIO4        | левый глаз · SG90 / любой хобби-серво |
| Servo 2 PWM   | GPIO2        | правый глаз · SG90 / любой хобби-серво |

> GPIO2 у ESP32-C3 — strapping-пин. Обычный серво (высокоомный вход по сигналу)
> загрузке не мешает; если плата перестаёт стартовать — временно отключите серво
> на GPIO2 или перенесите его на другой свободный GPIO.
> Серво лучше питать от отдельного 5 В источника с общей землёй (GND) с ESP32-C3.

## Возможности

- 10 LED-анимаций (off, static, rainbow, rainbow_cycle, chase, breathing,
  theater, wipe, scanner, dual) + выбор основного и фонового цвета,
  плюс RGB-палитра с ползунками.
- Регулировка яркости (0–255) и скорости анимации (10–500 мс).
- 2 серво (лючки глаз): слайдеры, пресеты (0/45/90/135/180°), синхронный режим,
  вкл/выкл каждого канала, регулировка скорости сервоприводов и неблокирующие
  сценарии (подмигнуть левым/правым, моргнуть обоими, открыть, закрыть, центр,
  домой, стоп).
- Веб-интерфейс на WebSocket (порт 81) + HTTP (порт 80), встроен в PROGMEM.
- Точка доступа Wi-Fi (`Kuzmich`) + mDNS: `http://esp32-control.local`.
- Состояние сохраняется в NVS (переживает перезагрузку).
- OTA-обновление прошивки.
- Защита: rate-limit по WS и HTTP, лимит клиентов, watchdog, контроль heap.

## Особенности сборки под ESP32-C3

- Цель сборки → `board = esp32-c3-devkitm-1` (RISC-V, single-core).
- Serial-монитор идёт через встроенный USB-CDC
  (`-DARDUINO_USB_CDC_ON_BOOT=1`, `-DARDUINO_USB_MODE=1`).
- Пины: WS2812B → GPIO10, Servo 1 → GPIO4, Servo 2 → GPIO2.
- Серво включены по умолчанию (`-DENABLE_SERVOS=true`).
- `MAX_WS_CLIENTS` = 4 (у C3 меньше heap, чем у классического ESP32).
- LittleFS используется только для отдачи фото команды (`data/team.jpg`).

## Сборка (PlatformIO)

```bash
pio run -t upload                                   # по USB
pio run -t upload --upload-port esp32-control.local # по OTA (пароль: ota-esp32)
pio device monitor                                  # лог 115200
```

Заливка файловой системы с фото команды:

```bash
pio run -t uploadfs                                 # зальёт содержимое data/ (team.jpg)
```

## Подключение

1. Подключись к Wi-Fi точке `Kuzmich` (пароль `12345678`).
2. Открой `http://esp32-control.local` (или IP из монитора, обычно `192.168.4.1`).

## Настройка

Все пины/лимиты/дефолты — в `src/config.h` (или через `-D` флаги в
`platformio.ini`). Для режима клиента (STA) раскомментируй `STA_SSID`/`STA_PASS`
и подмени блок `WiFi.softAP` в `setupWiFi()`.
